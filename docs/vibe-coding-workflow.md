# **vibe-coding-workflow.md**

# **学生成果认定与综合测评管理平台**

Vibe Coding开发工作流

版本：V1.0

------

# **一、目标**

本项目采用：

```text
产品设计
+
Claude Code
+
人工验收
```

方式开发。

开发原则：

```text
小步快跑

一次只做一个功能

先运行

再优化
```

禁止：

```text
一次开发整个系统

一次修改几十个文件

不测试直接进入下一阶段
```

------

# **二、项目准备**

项目结构：

```text
saep/

├── docs/
│
├── backend/
│
├── frontend/
│
└── README.md
```

文档放入：

------

# **三、Claude Code工作模式**

不要这样提问：

```text
帮我开发整个系统
```

错误原因：

```text
任务太大

上下文容易丢失

代码质量不可控
```

------

正确方式：

```text
完成 TASK-001
```

一次只开发一个任务。

------

# **四、标准开发流程**

每个任务都遵循：

```text
需求

↓

开发

↓

运行

↓

测试

↓

验收

↓

提交
```

------

# **五、第1周开发计划**

目标：

完成基础框架。

------

TASK-001

创建Django项目

提示词：

请阅读：

docs/CLAUDE.md

完成TASK-001

要求：

1 创建Django项目

2 创建apps目录

3 配置DRF

4 配置开发环境

5 输出项目结构

不要开发其它功能

------

验收：

```bash
python manage.py runserver
```

能够启动。

------

TASK-002

JWT认证

提示词：

请阅读CLAUDE.md

完成TASK-002

要求：

1 使用Simple JWT

2 实现登录接口

3 实现刷新Token接口

4 实现获取当前用户接口

5 提供测试方法

------

验收：

Postman登录成功。

------

TASK-003

RBAC权限

提示词：

完成TASK-003

要求：

使用Django Group

创建：

student

evaluator

counselor

college_admin

super_admin

生成初始化脚本

------

验收：

后台能看到角色。

------

# **六、Claude开发规则**

每次开始前：

发送：

```text
请先阅读：

docs/CLAUDE.md

docs/01-system-design.md

docs/07-task-list.md

然后完成TASK-XXX
```

------

禁止：

```text
自己发挥

擅自新增功能

擅自修改数据库设计
```

------

# **七、Bug处理流程**

不要说：

```text
这里报错了怎么办
```

------

正确方式：

```text
出现错误：

<完整错误日志>

相关文件：

<代码>

请分析原因

给出修复方案

不要直接修改代码
```

------

先分析。

确认方案后：

```text
请按照方案修复
```

------

# **八、代码审查流程**

每完成一个TASK：

执行：

```text
请审查刚才开发的代码

检查：

1 是否符合CLAUDE.md

2 是否符合Django最佳实践

3 是否存在安全问题

4 是否存在冗余代码

5 给出优化建议
```

------
